虚拟主机与请求的分发

server_name（对应用户请求中的主机域名）并通过server块来定义虚拟主机。server块就是一个虚拟主机，它只处理与之相对应的主机域名请求。

sndbuf=size(设置监听句柄的SO_SNDBUF参数)/

  • 主机名称 server_name name[…]

    1
2

    nginx会取出header头中的Host，与每个server中的server_name进行匹配,决定到底由哪一个server块来处理这个请求。
全匹配>前缀通配符>后缀通配符>正则

  • server_name_in_redirect on|off 重定向主机名称的处理

    1

    on打开时，表示在重定向请求时会使用server_name里配置的第一个主机名代替原先请求中的Host头部。

  • location[=|~|~*|^~|@]/uri/{…}

    尝试根据用户请求中的URI来匹配上面的/uri表达式，
如果可以匹配，就选择location{}块中的配置来处理用户请求。

匹配方式(有顺序 )
=表示把URI作为字符串
~表示匹配URI时是字母大小写敏感的
~*表示匹配URI时忽略字母大小写问题
正则
location  /匹配所有请求

请求处理时内存与磁盘资源分配配置

  • client_body_in_file_only on|clean|off;(HTTP包体只存储到磁盘文件中,默认off)

  • client_body_in_single_buffer on|off;(HTTP包体尽量写入到一个内存buffer，默认off)

    1

    因为用户请求的HTTP包体大小超过client_body_buffer_size设置的值，包体还是会写入到磁盘文件中

  • client_header_buffer_size size;(存储HTTP头部的内存buffer大小)

    1
2

    定义了正常情况下Nginx接收用户请求中HTTPheader部分（包括HTTP行和HTTP头部）
时分配的内存buffer大小.

  • large_client_header_buffers number size;(存储超大HTTP头部的内存,buffer个数，buffer大小)

    1

    每一个header的大小超过单个buffer的大小，或者总和超过number * size 会报错

  • client_body_buffer_size size;(存储HTTP包体的内存buffer大小)

    1

    HTTP包——>缓存——>写入磁盘

  • connection_pool_size size;(TCP连接内存池初始大小)

  • request_pool_size size;(HTTP请求处理内存池)

网络连接与设置

  • client_header_timeout time（读取HTTP头部的超时时间 默认单位：秒 60）
  • client_body_timeout time (读取HTTP包体的超时时间 默认单位：秒 60)
  • send_timeout time (发送响应的超时时间,响应用户但用户一直没有接受数据包，超时会关闭链接)
  • reset_timeout_connection on|off (默认off，开启后连接超时后将通过向客户端发送RST包来直接重置连接，不是使用正常情形下的四次握手关闭TCP连接，而是直接向用户发送RST重置包，不再等待用户的应答，直接释放Nginx服务器上关于这个套接字使用的所有缓存 如TCP滑动窗口)
  • lingering_close off|on|always(默认on 控制Nginx关闭用户连接的方式，是否必须先连接上用户)
  • lingering_timeout time(发送的数据到达服务器超时时间)
  • keepalive_timeout time(keepalive超时时间)
  • keepalive_requests n;(默认100 一个keepalive长连接上允许承载的请求最大数)

文件操作优化配置

  • sendfile on|off(启用Linux的sendfile系统调用来发送文件,减少了内核态与用户态之间的两次内存复制)
  • aio on|off(启用Linux系统上内核级别的异步文件I/O功能。与sendfile互斥)
  • directio size|off(Linux系统上使用O_DIRECT选项去读取文件，大文件读取速度会有优化，缓冲区大小为size。与sendfile互斥)

  • open_file_cache max=N[inactive=time]|off(文件缓存，默认off，会缓存文件句柄,文件大小，上次修改时间…减少对磁盘的操作)

    1
2
3
4
5
6

    open_file_cache max=1000 inactive=20s;

max:表示在内存中存储元素的最大个数。达到最大限制数量后，将采用LRU（Least Recently
Used）算法进行淘汰。
inactive:在inactive指定的时间段内没有被访问过的元素将会被淘汰。
off:关闭缓存功能。

  • open_file_cache_errors on|off(是否缓存打开文件错误的信息)

  • open_file_cache_min_uses number(不被淘汰的最小访问次数,默认1)

  • open_file_cache_valid time(检验缓存中元素有效性的频率,默认60s)

其他

  • ignore_invalid_headers on|off(忽略不合法的HTTP头部，默认on)
  • underscores_in_headers on|off(HTTP头部是否允许下划线,默认off)
  • if_modified_sinceoff|exact|before
  • log_not_found on|off(文件未找到时是否记录到error日志)
  • resolver address…(DNS解析地址)
  • resolver_timeout time(DNS解析的超时时间，默认30s)
  • server_tokens on|off(返回错误页面时是否在Server中注明Nginx版本)

反向代理

反向代理:用代理服务器来接受Internet上的连接请求，然后将请求转发给内部网络中的上游服务器，并将从上游服务器上得到的结果返回给Internet上请求连接的客户端。

Nginx并不会立刻转发到上游服务器，而是先把用户的请求（包括HTTP包体）完整地接收到Nginx所在服务器的硬盘或者内存中，然后再向上游服务器发起连接，把缓存的客户端请求转发到上游服务器。[增加了一点请求处理时间,但降低了上游服务器的负载，把压力放在了Nginx上]

负载均衡

  • upstream name{…} (定义了一个上游服务器的集群，便于反向代理中的proxy_pass使用)
  • server nameparameters

server参数：

1
2
3
4
5

weight=num:上游服务器转发的权重。
max_fails=num(def 1) fail_timeout=time(def 10s)：表示该时间段内转发失败多少次后就认为上游服务器暂时不可用，用于优化反向代理功能。
down：表示所在的上游服务器永久下线，只在使用ip_hash配置项时才有用。
·backup：在使用ip_hash配置项时它是无效的。它表示所在的上游服务器只是备份服务器，
只有在所有的非备份上游服务器都失效后，才会向所在的上游服务器转发请求。

反向代理

  • proxy_pass URL

    1
2
3

    将当前请求反向代理到URL参数指定的服务器上，URL可以是主机名/IP:PROT/UNIX句柄/upstream块的形式。

proxy_pass http://localhost:8000/uri/;

  • proxy_method method (转发时的协议方法名)

  • proxy_hide_header the_header;(Nginx将上游服务器的响应转发给客户端，这里指定哪些HTTP Header 字段不能被转发，默认不会转发Server、X-Pad和X-Accel-*)

  • proxy_pass_header the_header;(允许转发的HTTP Header)

  • proxy_pass_request_body on|off(默认on，是否向上游服务器发送HTTP包体部分)

  • proxy_pass_request_headers on|off

  • proxy_redirect[default|off|redirect replacement]

    1
2
3

    上游服务器返回的响应是重定向或刷新请求，proxy_redirect可以重设HTTP头部的location或refresh字段。

proxy_redirect http://localhost:8000/free/ http://frontend/free/
