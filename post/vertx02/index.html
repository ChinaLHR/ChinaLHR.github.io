Web组件

Router

Router 接收 HTTP 请求，并查找首个匹配该请求的 Route，然后将请求传递给这个 Route。每一个被路由的请求对应一个唯一的 RoutingContext，这个实例会被传递到所有处理这个请求的处理器上。

class MainVerticle : AbstractVerticle() {

  override fun start(startFuture: Future<Void>) {
    val server = vertx
      .createHttpServer()

    val router = Router.router(vertx)
    router.route("/hello").handler {
      val response = it.response()
      response.putHeader("content-type", "text/plain")
      response.end("Hello World from Vert.x-Web!")
    }

    server.requestHandler(router).listen(8000)
  }

}

//精确路径
router.route().path("/some/path/")
//路径前缀
router.route().path("/some/path/*")
//捕捉路径参数:
Route route = router.route(HttpMethod.POST, "/catalogue/products/:producttype/:productid/");

route.handler(routingContext -> {

  String productType = routingContext.request().getParam("producttype");
  String productID = routingContext.request().getParam("productid");

  // 执行某些操作...
});

//正则表达式路由
router.route().pathRegex(".*foo");

//HTTP Method路由(不指定路径默认拦截全部)
Route route = router.route(HttpMethod.POST, "/some/path/");

//consumes请求媒体类型（MIME types）路由
router.route().consumes("text/html").handler(routingContext -> {
  // 所有 `content-type`消息头的值为的请求会调用这个处理器`text/html`，可以匹配多个，也可以模糊匹配
});

//produces响应MIME类型
router.route().produces("application/json")

  • 子路由 目的：分隔为多个 Router，达到复用handler

实现：通过将一个 Router 挂载到另一个 Router 的挂载点上。挂载的 Router 被称为子路由（Sub Router）。Sub router 上也可以挂载其他的 sub router。

RoutingContext

在请求的生命周期中，您可以通过路由上下文 RoutingContext 来维护任何您希望在处理器之间共享的数据。
