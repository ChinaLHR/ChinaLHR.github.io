ngx_http_core_module模块

虚拟主机与请求的分发

server_name（对应用户请求中的主机域名）并通过server块来定义虚拟主机。server块就是一个虚拟主机，它只处理与之相对应的主机域名请求。

  • 监听端口 listen
listen address:port 其他参数
其他参数：default_server(默认server块)/
backlog=num(表示TCP中backlog队列的大小)/
rcvbuf=size(设置监听句柄的SO_RCVBUF参数)/
sndbuf=size(设置监听句柄的SO_SNDBUF参数)/
ssl(在当前监听的端口上建立的连接必须基于SSL协议)
  • 主机名称 server_name name[…]
nginx会取出header头中的Host，与每个server中的server_name进行匹配,决定到底由哪一个server块来处理这个请求。
全匹配>前缀通配符>后缀通配符>正则
  • server_name_in_redirect on|off 重定向主机名称的处理
on打开时，表示在重定向请求时会使用server_name里配置的第一个主机名代替原先请求中的Host头部。
  • location[=|~|~*|^~|@]/uri/{…}
尝试根据用户请求中的URI来匹配上面的/uri表达式，
如果可以匹配，就选择location{}块中的配置来处理用户请求。

匹配方式(有顺序 )
=表示把URI作为字符串
~表示匹配URI时是字母大小写敏感的
~*表示匹配URI时忽略字母大小写问题
正则
location  /匹配所有请求

文件路径

  • root path (默认root html) 和alias path

root可以放置到http、server、location或if块中，而alias只能放置到location块

# 个请求的URI是/download/test.html，那么Web服务器将会返回服务器上/opt/web/html/download/test.html文件的内容

location /download/ {
   root /opt/web/html/;
}

# alias写法
location /download/ {
   alias /opt/web/html/download;
}
  • 首页 index file…
location / {
    root   path;
    index /index.html /html/index.php /index.php;
}
  • 根据HTTP返回码重定向页面(error_page code[code…][=|=answer-code]uri|@named_location)
error_page   404          /404.html;

# 更改错误码
error_page 404 =200 /empty.gif;

重定向到另一个location中进行处理
location / (
    error_page 404 @fallback;
)
location @fallback (
    proxy_pass http://backend;
)

请求处理时内存与磁盘资源分配配置

  • client_body_in_file_only on|clean|off;(HTTP包体只存储到磁盘文件中,默认off)
  • client_body_in_single_buffer on|off;(HTTP包体尽量写入到一个内存buffer，默认off)
因为用户请求的HTTP包体大小超过client_body_buffer_size设置的值，包体还是会写入到磁盘文件中
  • client_header_buffer_size size;(存储HTTP头部的内存buffer大小)
定义了正常情况下Nginx接收用户请求中HTTPheader部分（包括HTTP行和HTTP头部）
时分配的内存buffer大小.
  • large_client_header_buffers number size;(存储超大HTTP头部的内存,buffer个数，buffer大小)
每一个header的大小超过单个buffer的大小，或者总和超过number * size 会报错
  • client_body_buffer_size size;(存储HTTP包体的内存buffer大小)
HTTP包——>缓存——>写入磁盘
  • connection_pool_size size;(TCP连接内存池初始大小)
  • request_pool_size size;(HTTP请求处理内存池)
